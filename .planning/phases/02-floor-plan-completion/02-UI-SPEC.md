# Phase 2 -- UI Design Contract

> Visual and interaction contract for floor plan completion features: knock-to-enter banner, offline fade-out, default space admin UI, and reconnection toasts. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | new-york |
| Component library | radix |
| Icon library | lucide-react |
| Font | System default (no custom font loaded -- inherits browser sans-serif) |
| Themes | 4 themes: neon, zen, obsidian, paper (all CSS-variable-driven via `tokens.css`) |
| CSS framework | TailwindCSS 4.2.1 with `tailwindcss-animate` plugin |

**Note:** This project uses a multi-theme architecture. All color references in this contract use CSS custom property names (e.g., `--vo-signal-success`) rather than hardcoded hex values. Each theme defines its own values for these properties in `src/styles/themes/tokens.css`. Components MUST use `var(--vo-*)` tokens or shadcn semantic classes (`text-foreground`, `bg-card`, etc.) to remain theme-compatible.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status dot offset |
| sm | 8px | Compact element spacing, knock banner internal gaps, button gaps |
| md | 16px | Default element spacing, card padding (`p-4`), section padding |
| lg | 24px | Section padding, neighborhood section gaps (`--vo-neighborhood-section-gap`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (not expected in this phase) |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Knock banner action buttons: 36px touch target (`w-9 h-9`) -- matches existing KnockToast pattern
- Avatar size: 36px (`--vo-avatar-size`) -- established in tokens.css, not on 8-point scale
- Detail panel padding: 16px (`--vo-detail-panel-padding`) -- established token
- Detail panel gap: 12px (`--vo-detail-panel-gap`) -- established token, 4px-aligned

## Typography

This phase does not introduce new typography roles. All text uses the existing Tailwind utility classes already established in the floor plan components. Exactly 2 weights are used: 400 (`font-normal`) and 600 (`font-semibold`).

| Role | Tailwind Class | Approx Size | Weight | Line Height | Usage in Phase 2 |
|------|---------------|-------------|--------|-------------|-------------------|

Weights used: 400 (`font-normal`), 600 (`font-semibold`). No other font weights are permitted in this phase.

## Visual Focal Point

**Neutral state (no active knock, no user activity):** The primary visual anchor is the user's current active space card, which renders with an accent border (`--vo-active-border`) distinguishing it from other space cards. When the user is not in any space, the floor plan grid itself (the collection of space cards organized by neighborhood) serves as the focal point, with the largest or first neighborhood section drawing the eye via its section heading.

**Active knock state:** The KnockBanner at the top of the relevant space card becomes the focal point, elevated by its success-tinted background and slide-in animation.

## New UI Elements Contract

### 1. KnockBanner (FLOR-01) -- NEW component

Replaces KnockToast for occupant notifications. Rendered inside ModernSpaceCard at the top.

**Layout:**
```
+--[Space Card]---------------------------------------------+
| +--[KnockBanner]----------------------------------------+ |
| | [Avatar 40px] [Text block        ] [Approve] [Deny]  | |
| |                "X is knocking"                         | |
| |                "Requesting access"                     | |
| +-------------------------------------------------------+ |
| [normal card content below]                                |
+------------------------------------------------------------+
```

**Specifications:**
- Position: First child inside card content area, full width of card minus card padding
- Background: `bg-[var(--vo-signal-success)]/10` (subtle success tint, theme-aware)
- Border: `border border-[var(--vo-signal-success)]/30` with `rounded-lg`
- Padding: `p-3` (12px) -- compact to avoid card height explosion
- Layout: `flex items-center gap-3`
- Avatar: 40px circle (`w-10 h-10 rounded-full`), fallback shows `DoorOpen` icon from lucide
- Text block: `flex-1 min-w-0`
  - Primary: `text-sm font-semibold text-foreground` for requester name + `text-sm font-normal text-foreground` for " is knocking"
  - Secondary: `text-xs font-normal text-muted-foreground` -- "Requesting access to this space"
- Approve button: `w-9 h-9 rounded-lg bg-[var(--vo-signal-success)]/20 hover:bg-[var(--vo-signal-success)]/30 text-[var(--vo-signal-success)]` -- `Check` icon (lucide)
- Deny button: `w-9 h-9 rounded-lg bg-[var(--vo-signal-critical)]/20 hover:bg-[var(--vo-signal-critical)]/30 text-[var(--vo-signal-critical)]` -- `X` icon (lucide)
- Click-stop: `data-avatar-interactive="true"`, `onClick={e.stopPropagation()}`, `onPointerDown={e.stopPropagation()}`
- Accessibility: `role="alert"`, `aria-live="polite"`, `aria-atomic="true"`
- Sound: `knock.mp3` plays via `new Audio()` when banner appears (already established pattern from Phase 1)
- Animation: Slide down from top -- `animate-in slide-in-from-top-2 duration-300` (tailwindcss-animate)
- Reduced motion: Opacity fade only, no slide (`prefers-reduced-motion: reduce`)

### 2. Knock Button on Space Card (FLOR-01) -- EXTEND ModernSpaceCard

**Placement:** In the card footer area, next to existing action buttons. Visible only on restricted spaces when user is NOT an occupant.

**Specifications:**
- Button variant: shadcn `Button` with `variant="outline"` and `size="sm"`
- Icon: `DoorClosed` (lucide) when idle, `Loader2` (lucide, spinning) when knocking
- Label: "Knock" (idle), "Knocking..." (in-progress), "Denied" (denied), "Wait {seconds}s" (cooldown)
- States:
  - `idle`: `variant="outline"` with normal styling
  - `knocking`: `variant="outline"` with `animate-pulse`, disabled
  - `denied`: `variant="destructive"` with `opacity-60`, disabled for 60s
  - `cooldown`: `variant="ghost"` showing countdown, disabled
  - `approved`: Button disappears (user auto-joins space)
- `data-avatar-interactive="true"` on button wrapper
- `e.stopPropagation()` on `onClick` and `onPointerDown`
- Minimum touch target: 36px height (inherent in `size="sm"`)

### 3. Offline Fade-Out Animation (FLOR-02) -- EXTEND UserAvatarPresence + AvatarGroup

**CSS Animation (add to tokens.css):**
```css
@keyframes vo-avatar-offline-fade {
  from { opacity: 1; }
  to { opacity: 0; }
}

.vo-avatar-offline-fade {
  animation: vo-avatar-offline-fade 3s ease-out forwards;
  pointer-events: none;
}

@media (prefers-reduced-motion: reduce) {
  .vo-avatar-offline-fade {
    animation-duration: 0.3s;
  }
}
```

**Behavior:**
- When user status changes to `offline`: apply class `vo-avatar-offline-fade` to avatar wrapper
- During fade: `pointer-events: none` prevents interaction with departing avatar
- After 3s animation completes: remove user from DOM (cleanup via `onAnimationEnd` callback)
- Only `offline` triggers fade -- `away` users remain fully visible with yellow status dot
- Existing `vo-avatar-leave` animation (600ms, includes scale) is NOT reused here -- the offline fade is slower (3s) and opacity-only to match the "subtle but noticeable" intent from CONTEXT.md

## Interaction States

### Knock Button State Machine (visual mapping)

| State | Button Label | Button Style | Disabled | Icon |
|-------|-------------|-------------|----------|------|
| `idle` | "Knock" | `variant="outline"` | No | `DoorClosed` |
| `knocking` | "Knocking..." | `variant="outline" opacity-70` | Yes | `Loader2` (spinning) |
| `approved` | (hidden) | N/A -- user auto-joins | N/A | N/A |
| `denied` | "Denied" | `variant="destructive" opacity-60` | Yes | `ShieldX` |
| `timeout` | "No response" | `variant="ghost" opacity-60` | Yes | `Clock` |
| `cooldown` | "Wait {N}s" | `variant="ghost"` | Yes | `Timer` |

### Knock Banner Lifecycle

1. Knock request arrives via postgres_changes INSERT on knock_requests
2. `knock.mp3` plays via `new Audio('/sounds/knock.mp3')`
3. KnockBanner slides in at top of relevant space card (the card the occupant is currently in)
4. Occupant clicks Approve or Deny
5. On Approve: banner slides out, requester auto-joins, approval toast shown to requester
6. On Deny: banner slides out, denial toast shown to requester
7. If no response within 30s: banner auto-dismisses, timeout toast shown to requester

### Offline Fade-Out Lifecycle

1. User status changes to `offline` (detected via postgres_changes on users table OR Supabase Presence leave event)
2. `vo-avatar-offline-fade` class applied to avatar wrapper
3. Avatar fades from `opacity: 1` to `opacity: 0` over 3 seconds
4. `pointer-events: none` active during fade to prevent stale interactions
5. After animation completes (`onAnimationEnd`): remove user from rendered list
6. Server-side: `users.current_space_id` cleared, `space_presence_log.exited_at` updated

### Reconnection Flow (visual states)

1. Page loads -- check `localStorage` for `vo-disconnect-timestamp` and `lastSpaceId`
2. If within 5-minute grace: auto-join last space silently, show "Reconnected to [Space]" toast
3. If last space is full: fallback to home space, show "[Space] is full -- moved to [Home]" toast
4. If grace expired: treat as fresh login, join home space or default space (no toast unless first-time)
