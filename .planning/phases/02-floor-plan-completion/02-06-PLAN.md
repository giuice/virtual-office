**1. Convert the company update API to App Router format**

Create `src/app/api/companies/update/route.ts` as a proper App Router route handler:

```typescript
import { NextResponse } from 'next/server';
import { ICompanyRepository } from '@/repositories/interfaces';
import { SupabaseCompanyRepository } from '@/repositories/implementations/supabase';
import { createSupabaseServerClient } from '@/lib/supabase/server-client';
import type { Company } from '@/types/database';

export const dynamic = 'force-dynamic';

export async function PATCH(request: Request) {
  try {
    const supabase = await createSupabaseServerClient();

    // Authenticate the caller
    const { data: authData, error: authError } = await supabase.auth.getUser();
    if (authError || !authData.user) {
      return NextResponse.json({ error: 'Authentication required' }, { status: 401 });
    }

    const url = new URL(request.url);
    const id = url.searchParams.get('id');
    if (!id) {
      return NextResponse.json({ error: 'Missing id parameter' }, { status: 400 });
    }

    const companyData: Partial<Omit<Company, 'id' | 'createdAt' | 'updatedAt'>> = await request.json();

    const companyRepository: ICompanyRepository = new SupabaseCompanyRepository(supabase);
    const updatedCompany = await companyRepository.update(id, companyData);

    if (!updatedCompany) {
      return NextResponse.json({ success: false, message: 'Company not found or update failed' }, { status: 404 });
    }

    return NextResponse.json({
      success: true,
      company: updatedCompany,
      message: 'Company updated successfully',
    });
  } catch (error) {
    console.error('Error updating company:', error);
    return NextResponse.json({
      success: false,
      error: error instanceof Error ? error.message : 'Failed to update company',
    }, { status: 500 });
  }
}
```

Then DELETE the old `src/app/api/companies/update.ts` file. It uses Pages Router format (`export default handler` with `NextApiRequest`/`NextApiResponse`) which Next.js App Router does not register, causing the 404 HTML response.

IMPORTANT: The client in `src/lib/api.ts` already calls `fetch('/api/companies/update?id=...')` with `method: 'PATCH'`, so the new route handler matches the existing client contract exactly. No client changes needed.

**3. Fix settings merge in CompanyContext.tsx**

In `src/contexts/CompanyContext.tsx`, in the `updateCompanyDetails` function (around line 275-279), change the settings merge:
```typescript
// BEFORE:
const mergedData: Partial<Company> = {
  ...data,
  settings: data.settings
    ? { ...company.settings, ...data.settings }
    : data.settings,
};
// AFTER:
const mergedData: Partial<Company> = {
  ...data,
  settings: data.settings !== undefined
    ? { ...company.settings, ...data.settings }
    : undefined,
};
```

When `data.settings` is `undefined` (e.g., saving company name only), the old code passes `settings: undefined` to the API, which could NULL-wipe the settings column. The fix omits the `settings` key entirely from `mergedData` when the caller didn't provide it, so the API only updates the fields that were actually passed.

Also update the optimistic state update (around lines 292-295) to use the same guard:
```typescript
settings: mergedData.settings !== undefined
  ? { ...prev.settings, ...mergedData.settings }
  : prev.settings,
```
