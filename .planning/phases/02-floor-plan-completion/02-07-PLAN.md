Two changes in different files, both needed for reload to work correctly:

**1. Fix FloorPlan placement useEffect (floor-plan.tsx lines 263-279)**

The current useEffect calls `handleEnterSpace(space)` which is UI-only -- it sets `selectedSpace` and `highlightedSpaceId` but does NOT call the location API. This means after reload, the user appears "in" a space locally but the DB still has `currentSpaceId: null`, so other clients never see them.

Replace the entire useEffect block (lines 263-279) with a visual-selection-only version that does NOT call handleEnterSpace:

```typescript
// Hydrate visual selection on load -- useLastSpace handles the actual API placement
useEffect(() => {
  if (!currentUserProfile || spaces.length === 0) {
    return;
  }

  // If user is already in a space (from presence or useLastSpace API call), highlight it
  const targetSpaceId = currentSpaceId
    || getReconnectionContext(currentUserProfile, spaces, company, lastSpaceId).spaceId;
  if (!targetSpaceId) {
    return;
  }

  const space = spaces.find((candidate) => candidate.id === targetSpaceId);
  if (space) {
    // Only set visual state -- useLastSpace handles the API call to /api/users/location
    setSelectedSpace(space);
    setHighlightedSpaceId(space.id);
  }
}, [company, currentSpaceId, currentUserProfile, lastSpaceId, spaces]);
```

The critical difference: no `handleEnterSpace(space)` call, no `saveLastSpace(space.id)` call. The `useLastSpace` hook already calls `/api/users/location` via its own useEffect, which updates the DB. FloorPlan should only sync the visual highlight.

**2. Add last_active timestamp as grace-rejoin backup in enforceSpaceAuthorization (route.ts)**

The current grace-rejoin check in `enforceSpaceAuthorization` depends solely on `exited_at` from `space_presence_log`. But on reload, the beacon POST that writes `exited_at` may arrive AFTER the rejoin PUT (race condition), making `getMostRecentPriorOccupancy` return null and causing a false 403.

Add a secondary grace check using the user's `last_active` timestamp from the `users` table. After the `getMostRecentPriorOccupancy` call (around line 242), add:

```typescript
const priorOccupancy = await getMostRecentPriorOccupancy(supabase, spaceId, authenticatedUser.id);

// Primary check: exited_at from space_presence_log
const hasGraceRejoinByExitLog = Boolean(
  priorOccupancy?.exited_at &&
    Date.now() - new Date(priorOccupancy.exited_at).getTime() < SPACE_REJOIN_GRACE_MS
);

// Secondary check: last_active from users table (handles beacon POST race)
// If user was recently active AND had this space as their previous space,
// they are likely rejoining after a reload where the beacon hasn't committed yet
const hasGraceRejoinByLastActive = Boolean(
  !hasGraceRejoinByExitLog &&
    authenticatedUser.lastActive &&
    Date.now() - new Date(authenticatedUser.lastActive).getTime() < SPACE_REJOIN_GRACE_MS
);

// Also check if there is an OPEN (no exited_at) presence log for this user+space
// This handles the case where the user was in the space and reloaded before beacon
let hasOpenPresenceLog = false;
if (!hasGraceRejoinByExitLog && !hasGraceRejoinByLastActive) {
  const { data: openLog } = await supabase
    .from('space_presence_log')
    .select('id')
    .eq('user_id', authenticatedUser.id)
    .eq('space_id', spaceId)
    .is('exited_at', null)
    .limit(1)
    .maybeSingle();
  hasOpenPresenceLog = Boolean(openLog);
}

if (hasGraceRejoinByExitLog || hasGraceRejoinByLastActive || hasOpenPresenceLog) {
  return { authorizedByUserId: null, consumedKnockRequestId: null };
}
```

Replace the existing `hasGraceRejoin` variable and its associated `if (hasGraceRejoin)` block with this expanded version. Note that `authenticatedUser` is of type `User` from `@/types/database`, which includes `lastActive?: string` from the users table's `last_active` column.
