## Trust Boundaries

| Boundary | Description |
|---|---|
| Auth claims -> database identity | JWT identity must map to the application user before application foreign keys are compared. |
| Privileged RPC -> Presence authority | RPC execution can bypass ordinary RLS and must reauthorize under locks. |
| Browser topic -> realtime.messages | Untrusted topic/extension data crosses tenant and space isolation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-01 | Tampering / Denial of Service | concurrent lease claims | high | mitigate | Deterministic lock order, short transactions, unique per-space authority, post-lock checks, and later repeated real concurrency tests. |
| T-03-02 | Spoofing / Elevation of Privilege | observed RPCs | high | mitigate | Map JWT sub through users.supabase_uid, require exact sessions/occupancy/revisions, and expose only narrow service-role execution. |
| T-03-03 | Information Disclosure | private media topic | high | mitigate | Exact topic parser and four extension-specific policies fail closed; local catalog/RLS readback is blocking. |
</threat_model>
